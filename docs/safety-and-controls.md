# Seguridad y controles

## Controles principales

1. El workflow no debe inventar fuentes.
2. El árbitro no debe usar claims no soportados como base de veredicto.
3. El debate debe separar hechos, supuestos, inferencias y opiniones.
4. Las salidas deben guardar trazabilidad en Markdown y JSON.
5. Temas financieros, legales, médicos o laborales requieren revisión humana.
6. Las posturas deben generarse de forma independiente antes del contraexamen.
7. El reporte debe indicar qué evidencia cambiaría el veredicto.

## Riesgos

- Persuasión sin verdad.
- Falso consenso entre agentes.
- Sobreconfianza.
- Sesgo por formulación de la tesis.
- Evidencia web de baja calidad.
- Uso excesivo de tokens.

## Mitigaciones

- Auditoría de evidencia obligatoria.
- Crítica lógica obligatoria.
- Detección de sesgos obligatoria.
- Veredicto conservador cuando la evidencia sea débil.
- Primera corrida siempre acotada.
